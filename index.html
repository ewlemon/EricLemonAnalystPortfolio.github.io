AI Analyst Copilot Agent

The problem
Business teams always have one more question for the database, and nobody wants to wait a day for an analyst to run it. But handing an AI direct access to your data raises some obvious trust issues.
The approach
I built an agent on the Claude API with three tools: it can inspect the schema, run SQL, and draw charts. Two separate guardrails (a validator that only allows SELECT statements, plus a connection opened in read-only mode) make writes impossible. Every answer shows the exact SQL it ran and a short note on caveats.
The outcome
A live agent that answers questions like “which product categories are most profitable?” in seconds, chart included. I re-ran every example query myself to confirm the numbers matched. The point isn't replacing the analyst; it's giving them a faster first draft.

T-SQLETLSQL Server

Business SQL Project

The problem
A SaaS company's data lands as six messy CSV exports, about 98K rows covering customers, revenue, product usage, sales pipeline, subscriptions, and support tickets. The IDs have prefixes, every column is text, and somewhere in there is the answer to where revenue is leaking.
The approach
I staged everything as raw text first, stripped the ID prefixes, typed each column safely with TRY_CAST, and built reusable views on the clean tables so every analysis could start from the same solid base.
The outcome
Average customer lifetime value came out to $21.6K, with active customers worth nearly double churned ones. Starter-plan churn runs 2.69% against 1.18% for Enterprise, and the sales pipeline converts at just 5.6% with 67% of deals never reaching a demo. Three findings, three levers the business can pull.

PythonRegressionGitHub ActionsExcel

U.S. Stock Market Index Analysis

The problem
Everyone says the market grows over time, but which model actually fits that growth best? And could I build the analysis once and have it keep itself up to date?
The approach
I modeled five major U.S. indexes with Linear, Ridge, Lasso, and Degree-3 Polynomial regression, compared them on R² and RMSE, and added rolling-volatility views across the dot-com bust, 2008, and COVID. A GitHub Actions pipeline pulls fresh data every day and writes the results into an Excel workbook.
The outcome
The pipeline has run hands-off since launch, over 230 commits of iteration later. Model comparisons, closing values, and daily returns stay current without me touching a thing.

TableauDashboard DesignData Storytelling

Career Trends Dashboard

The problem
Career and salary stats are some of the easiest data to misread, and the decision they inform is a big one. I wanted a dashboard that was honest before it was pretty.
The approach
The dashboard projects 10-year outlooks: annual openings, growth, turnover, wages, and required education. The filters went through several rounds before they felt natural, and just as much work went into deciding what to leave out as what to include.
The outcome
One view where you can compare career paths on the numbers that matter, without the presentation nudging you toward a bad read. It's embedded right on this site and published on Tableau Public.
